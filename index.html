A D3 Implementation of SYMAP Interpolation Algorithms and Graphic Conventions

A D3 implementation of the overprinting technique used by early automated cartography platforms to render shaded areas on line printers. It also makes use of an in-browser interpolation function that, taking areal data as its input, generates an isopleth map. More specifically, it uses Donald Shepard's 1968 inverse distance weighting algorithm, developed for use in the SYMAP mapping platform at the Harvard Laboratory for Computer Graphics. For the curious, the paper in which Shepard first detailed his algorithm can be found here.

Unemployment in Kentucky, 2000-2013

A chorpleth map that displays unemployment in Kentucky counties between 2000 and 2013. The map is coded in Leaflet with some D3 flourishes. The data is loaded dynamically with an AJAX request, and classifications are calculated at run-time with the client. As the user mouses over a given county, a info-graphic displaying unemployment over time is dynamically generated using D3 and displayed on axes whose range is continuously recalculated based on the input domain. The map uses an HTML5 range slider to sequence through the years and update the map.

  • Source code on GitHub.
  • Data: American Community Survey

Hydroelectric, Solar, and Coal Power Plants In the United States

A proportional symbol map, coded using Leaflet, that displays all hydroelectric, solar, and coal power plants in the United States scaled by their total power output. When the user clicks a location, the map calculates the total power generated by each types of plants in a circular surrounding area and binds appropriate text to a popup that is redrawn with each user click. A double-click does the same with a more confined search area.
